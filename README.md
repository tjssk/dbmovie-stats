# 豆瓣观影记录统计工具

通过豆瓣 uid 采集"看过"列表，自动统计导演、类型、国家/地区、语言、上映年份等维度的分布，生成可视化报告。

## 功能

- **数据采集**：通过 uid 翻页获取"看过"列表，调用豆瓣 API 获取每部电影的导演、演员、类型、国家、上映年份、片长、豆瓣评分，从列表页信息行提取语言
- **统计分析**：导演 Top 30、导演评分对比（你的评分 vs 豆瓣）、类型/国家/语言分布、上映年份/年代分布、评分分布、观影时间线、AI 内容标签词云
- **AI 内容标签分析**（可选）：用你自己的 OpenAI 兼容 API 分析电影深层主题（女性/儿童/乡村/宗教/阶级等），生成词云图
- **可视化报告**：马卡龙低饱和配色，ECharts 图表（柱状图、饼图、折线图、词云图）
- **完整影片列表**：底部含可折叠的影片列表，支持按导演/演员/类型/国家/语言/标签/年份/评分任意组合筛选
- **导演悬停查看影片**：最常观看导演图表悬停时显示该导演你看过的所有作品
- **CSV 导出**：一键导出包含 AI 标签的完整数据
- **缓存机制**：已采集的电影详情缓存到本地，下次运行自动跳过
- **断点续传**：采集中断后可从断点继续
- **响应式设计**：电脑/平板/手机三档断点适配

## 安装

```bash
# 1. 创建虚拟环境
python -m venv .venv

# 2. 激活虚拟环境
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate

# 3. 安装依赖
pip install -r requirements.txt
```

## 运行

```bash
python app.py
```

打开浏览器访问 **http://localhost:5000**

## 使用流程

1. 在首页输入你的豆瓣 uid
2. （可选）如果你的观影记录是私密的，粘贴登录 Cookie
3. 点击"开始采集"，等待采集完成（200 部约需 5-8 分钟）
4. 点击"查看统计报告"查看可视化图表
5. （可选）展开"AI 内容标签分析"，填入你的 OpenAI 兼容 API 信息，批量分析电影深层主题
6. 在底部"完整影片列表"中按任意维度筛选
7. 可点击"导出 CSV"下载原始数据

## AI 内容标签分析

该功能使用用户自己的 AI 算力，**不会通过本工具服务器中转**：
- **API URL**：OpenAI 兼容接口地址，如 `https://api.openai.com/v1/chat/completions`
- **API Key**：直接存储在浏览器 localStorage，不上传到任何服务器
- **模型**：如 `gpt-4o-mini` / `claude-3-5-sonnet` / `deepseek-chat` 等
- **每批电影数**：建议 5-15，越大越快但 token 消耗越多

分析候选标签池：女性、儿童、青少年、老年、家庭、母亲、爱情、友情、孤独、成长、青春、死亡、救赎、复仇、自由、反叛、压抑、治愈、悬疑、惊悚、心理、犯罪、战争、政治、宗教、信仰、文化、传统、乡土、城市、乡村、自然、动物、奇幻、科幻、魔幻、童话、史诗、传记、历史、现实主义、梦境、实验、文艺、公路、旅行、教育、校园、职场、医疗、阶级、贫富、移民、种族、LGBTQ、残障、喜剧、悲剧、人性、欲望、记忆、身份、命运、底层、上流、知识分子、流浪汉、孤儿 等。

## 如何获取 uid

打开你的豆瓣电影主页，URL 中的标识即为 uid。
例如 `movie.douban.com/people/你的uid/` 中的 `你的uid`。

## 如何获取 Cookie（可选）

1. 在浏览器中登录豆瓣
2. 按 F12 打开开发者工具 → Application（应用）→ Cookies → `.douban.com`
3. 复制 `dbcl2` 和 `bid` 的值，格式如：`dbcl2="123456789:abc"; bid="xyz"`

如果你的观影记录是公开的，无需 Cookie。

## 数据来源

| 数据 | 来源 | 字段 |
|------|------|------|
| 观影记录 | collect 列表页 | 电影标题、用户评分、观影日期、短评 |
| 电影详情 | subject_abstract API | 导演、演员、类型、国家/地区、上映年份、片长、豆瓣评分 |
| 语言 | 列表页 intro 行 | 语言（通过已知语言列表匹配） |
| 内容标签 | 用户配置的 AI API | 深层主题标签（女性/儿童/宗教/阶级等） |

## 文件结构

```
douban-movie-stats/
├── app.py              # Flask Web 应用（含 AI 分析端点）
├── douban_spider.py    # 数据采集模块
├── analyzer.py         # 统计分析模块（含词云和影片列表聚合）
├── requirements.txt    # Python 依赖
├── templates/
│   ├── index.html      # 首页（输入 uid + 采集进度）
│   └── report.html     # 报告页（ECharts 可视化 + 筛选）
├── static/
│   └── css/style.css   # 薄荷巧克力主色 + 马卡龙图表色
└── data/               # 采集数据缓存（自动生成）
    └── {uid}/
        ├── watched.json      # 观影记录
        └── movies_cache.json # 电影详情缓存（含 AI 标签）
```

## 用途

1. **统计个人喜好**：了解自己的观影偏好（最爱导演、类型、国家等）
2. **辅助论文选题**：通过数据分析确定研究方向（如某类影片的观看频率、某地区影片的覆盖率）
3. **查漏补缺**：发现被忽略的影片范围、导演、类型，拓展观影视野
